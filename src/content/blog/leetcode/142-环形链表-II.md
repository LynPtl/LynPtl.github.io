# 142. 环形链表 II

> <strong>题目链接：</strong>[142. 环形链表 II](https://leetcode.cn/problems/linked-list-cycle-ii/)

## 题目描述

给定一个单向链表的头节点 <code>head</code>，返回链表开始入环的第一个节点。如果链表无环，则返回 <code>null</code>。

## 核心思考：快慢指针与数学推导

这道题是经典的“验证环形链表并寻找入口”问题，主要考察了<strong>双指针（快慢指针）</strong>技巧以及背后的数学原理。求解过程分为明确的两步：

### 1. 判断是否有环
定义两个指针：快指针 <code>p1</code> 每次向前走两步，慢指针 <code>p2</code> 每次向前走一步。如果链表存在环，这两个指针必定会在环内的某个节点相遇。如果在遍历过程中 <code>p1</code> 或者 <code>p1.next</code> 变为空，说明提前走到了链表尽头，可以直接判定为无环。

### 2. 定位环的入口位置
这是一道典型带有推导过程的数学题。假设链表头节点到环入口的距离为 <code>$L$</code>，环入口到快慢指针相遇点的距离为 <code>$k$</code>，环的总周长为 <code>$C$</code>。
当快慢指针相遇时：
- 慢指针 <code>p2</code> 走过的路程为：<code>$L + k$</code>
- 快指针 <code>p1</code> 走过的路程为：<code>$L + n \times C + k$</code>（<code>$n$</code> 为快指针已经绕环的圈数）

因为快指针的速度恒为慢指针的两倍，路程差也可得出如下等式：<code>$2(L + k) = L + n \times C + k$</code>
我们将等式稍微化简一下：<strong><code>$L = (n - 1) \times C + (C - k)$</code></strong>

<strong>这句话的直观含义是什么？</strong>
等式的左边 <code>$L$</code> 是从头节点走到环入口的距离；等式的右边指出，如果我们将指针从相遇点继续顺时针往下走 <code>$C-k$</code> 的距离（也就是走到环入口），再额外走完几整圈，总距离刚好等于 <code>$L$</code>。

因此，<strong>破局的方法</strong>就是：把其中一个指针重新移回链表头部 <code>head</code>，另一个指针依然停在相遇点。随后让两个指针<strong>同时每次只走一步</strong>。当头部的新指针走完 <code>$L$</code> 距离到达入口时，环内的指针也必然刚好“转完圈子”走到了入口处。两者相遇的地方，就是环入口。

## 解题思路 (Python)

``<code>python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, x):
#         self.val = x
#         self.next = None

class Solution:
    def detectCycle(self, head: Optional[ListNode]) -> Optional[ListNode]:
        p1 = head
        p2 = head
        while p1 != None and p1.next != None:
            p1 = p1.next.next
            p2 = p2.next
            if p1 == p2: 
                break
        if p1 == None or p1.next == None:
            return None
        p1 = head
        while p1 != p2:
            p1 = p1.next
            p2 = p2.next
        return p2
</code>`<code>

## 复杂度分析

- <strong>时间复杂度</strong>：$O(N)$，其中 $N$ 是链表节点的总数。在第一阶段判断是否有环时，两指针绕行最多不会超过线性次数；第二阶段各指针最多移动 $N$ 步即可相遇定位入口，总体的时间复杂度仍保持在线性级别。
- <strong>空间复杂度</strong>：$O(1)$。全程我们只额外分配了 </code>p1<code> 和 </code>p2` 两个常数级别的指针变量，没有产生其他任何额外的内存开销。

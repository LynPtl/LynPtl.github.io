# 238. 除了自身以外数组的乘积

> <strong>题目链接：</strong>[238. 除了自身以外数组的乘积](https://leetcode.cn/problems/product-of-array-except-self/)

## 题目描述

给你一个整数数组 <code>nums</code>，返回一个数组 <code>answer</code> ，其中 <code>answer[i]</code> 等于 <code>nums</code> 中除 <code>nums[i]</code> 之外其余各元素的乘积。

题目数据 <strong>保证</strong> 数组 <code>nums</code>之中任意元素的全部前缀元素和后缀（甚至是整个数组）的乘积都在  <strong>32 位</strong> 整数范围内。

<strong>请不要使用除法</strong>，且在 $O(n)$ 时间复杂度内完成此题。

## 核心思考：前缀积与后缀积的交汇

这道题的核心矛盾在于：不能使用除法。如果能用除法，直接计算总乘积再除以当前元素即可（只要处理好 0 的情况）。

在禁止除法的前提下，我们可以观察到：对于下标 <code>i</code> 的元素，其结果实际上是 <strong><code>i</code> 左侧所有元素的乘积</strong> 乘以 <strong><code>i</code> 右侧所有元素的乘积</strong>。

1. <strong>前缀积阵列 (<code>front</code>)</strong>：<code>front[i]</code> 存储 <code>nums[0...i-1]</code> 的乘积。
2. <strong>后缀积阵列 (<code>rear</code>)</strong>：<code>rear[i]</code> 存储 <code>nums[i+1...n-1]</code> 的乘积。
3. <strong>最终结果</strong>：<code>answer[i] = front[i] <em> rear[i]</code>。

这种思路将“除自身外的乘积”拆解为两个方向的线性扫描，避开了除法，同时完美符合 $O(n)$ 的时间要求。

## 解题思路 (Python)

``<code>python
class Solution:
    def productExceptSelf(self, nums: List[int]) -> List[int]:
        n = len(nums)
        answer = [0] </em> n
        front = [1] <em> n
        rear = [1] </em> n
        
        # 计算每个位置的前缀积
        for x in range(1, n):
            front[x] = front[x - 1] <em> nums[x - 1]
            
        # 计算每个位置的后缀积
        for x in range(n - 2, -1, -1):
            rear[x] = rear[x + 1] </em> nums[x + 1]

        # 前后缀积相乘即为结果
        for i in range(n):
            answer[i] = front[i] * rear[i]
            
        return answer
</code>`<code>

## 复杂度分析

- <strong>时间复杂度</strong>：$O(n)$，其中 $n$ 为数组长度。我们进行了三次独立的线性遍历（两次计算前后缀积，一次合并），总耗时与数组规模成正比。
- <strong>空间复杂度</strong>：$O(n)$。本解法使用了 </code>front<code> 和 </code>rear` 两个额外的辅助数组来存储中间结果。（注：可以通过直接在结果数组上操作来优化至 $O(1)$ 额外空间，但不计结果数组本身的情况下）。

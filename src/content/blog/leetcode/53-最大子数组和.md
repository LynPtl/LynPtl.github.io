# 53. 最大子数组和

> <strong>题目链接：</strong>[53. 最大子数组和](https://leetcode.cn/problems/maximum-subarray/)

## 题目描述

给你一个整数数组 <code>nums</code> ，请你找出一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。

<strong>子数组</strong>是数组中的一个连续部分。

## 核心思考：为什么相向双指针会失效？

在处理连续区间求和问题时，直觉上容易引入滑动窗口或相向双指针策略。然而，针对本题的数据特征（存在负数），我们需要进行严密的逻辑验算以确定数据结构和算法模式的适用性。

相向双指针（从两端向内收缩）的有效性前提，是能够基于当前状态，明确且单调地排除某一侧的非优区间。但在存在负数的序列中，区间和失去了<strong>单调性</strong>。

<strong>致命缺陷验算</strong>：
1. <strong>局部决策的不可判定性</strong>：例如 <code>nums = [2, -100, 50, -100, 3]</code>。即使两端起手都是正数，在向内收缩时，无法通过边界的局部大小比较，跨越深渊（<code>-100</code>）探测到被包裹在中间的全局最优解（<code>50</code>）。
2. <strong>边界崩溃 (Edge Case)</strong>：如果采取“两边收紧跳过负数”的优化策略，面对全负数数组 <code>nums = [-5, -2, -9]</code>，寻找非负数的逻辑会导致指针直接越界，无法返回合法子数组。

结论：基于区间向内收缩的双指针在此场景完全失效。我们需要将架构视角切换为<strong>单向流式遍历</strong>。

## 解题思路：动态规划与状态转移 (Kadane 算法)

摒弃区间两端的动态维护，我们转而关注“前序状态”对“当前元素”的数值影响。

在 $O(n)$ 的单向遍历中，假设当前访问元素为 <code>nums[i]</code>，并维护一个变量 <code>curr_sum</code> 记录 <code>nums[i]</code> 之前的连续子数组累加和。对于是否将 <code>nums[i]</code> 纳入现有的连续区间，取决于严格的数学关系：

<em> <strong>状态 A (<code>curr_sum > 0</code>)</strong>：<code>curr_sum + nums[i]</code> 必然大于单独的 <code>nums[i]</code>。前序状态提供了正向增益，应当继续累加。
</em> <strong>状态 B (<code>curr_sum < 0</code>)</strong>：<code>curr_sum + nums[i]</code> 必然小于 <code>nums[i]</code> 本身。前序的累加和对当前连续区间产生了<strong>数值上的负向拖累</strong>。此时应当果断丢弃前序历史，直接以当前的 <code>nums[i]</code> 作为新子数组的起点（另起炉灶）。

### 全局最优的锁定与边界防范

由于 <code>curr_sum</code> 仅代表<strong>局部最优</strong>（以当前元素结尾的最大子数组和），它会随着数组元素的正负交替不断波动。因此，必须引入一个独立的状态变量 <code>max_sum</code> 来持续捕获并锁定历史峰值。

<strong>边界防范 (Edge Case)</strong>：
针对全负数数组的极限输入（如 <code>[-5, -2, -9]</code>），如果 <code>max_sum</code> 初始化为 <code>0</code>，则会漏算真实的负数最大值。因此，必须将其底层状态初始化为负无穷大 <code>float('-inf')</code>，以确保极值捕获的绝对鲁棒性。

## 代码实现 (Python)

``<code>python
from typing import List

class Solution:
    def maxSubArray(self, nums: List[int]) -> int:
        # 全局峰值锁定，初始化为 -inf 防御全负数边界
        max_sum = float('-inf')
        # 局部连续和状态
        curr_sum = 0
        
        # O(N) 单向流式遍历
        for num in nums:
            # 状态转移：若前序累加和为负，则切断历史，消除负向拖累
            if curr_sum < 0:
                curr_sum = 0
                
            curr_sum = curr_sum + num
            
            # 动态更新全局最优解
            max_sum = max(curr_sum, max_sum)
            
        return max_sum
</code>`<code>

## 复杂度分析

- <strong>时间复杂度</strong>：$O(n)$，其中 $n$ 是数组 </code>nums<code> 的长度。我们只需要遍历一遍数组即可求得答案。
- <strong>空间复杂度</strong>：$O(1)$，我们仅使用常数级别的空间来存放 </code>curr_sum<code> 和 </code>max_sum` 这两个变量。

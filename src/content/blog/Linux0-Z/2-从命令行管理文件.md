# Linux 2 从命令行管理文件

关于文件系统基础、文件和目录操作、链接、以及I/O重定向和管道的核心概念。

---

## 1. 文件系统 (Filesystem) 基础

### 什么是文件系统？
- 文件系统是操作系统用来管理文件和目录的一种机制。
- 它控制着数据如何被存储和检索，实现对文件和目录的有序化、结构化管理。
- 常见的类Unix文件系统类型包括：<code>ext3</code>, <code>ext4</code>, <code>xfs</code>。

### <code>ls -l</code> 输出详解
执行 <code>ls -l</code> 命令可以查看文件和目录的详细属性，输出结果通常包含9列：

![VirtualBoxVM_2JATjTiWmo.png](https://pub-85d4dcece16844bf8290aa4b33608ccd.r2.dev/ShareX/2025/10/VirtualBoxVM_2JATjTiWmo.png)

1.  <strong>文件类型和权限</strong>: 第一个字符表示类型（<code>-</code> 普通文件, <code>d</code> 目录, <code>l</code> 符号链接），后面9个字符表示权限。
2.  <strong>硬链接数</strong>: 指向此文件inode的硬链接数量。
3.  <strong>所有者 (Owner)</strong>: 文件或目录的拥有者。
4.  <strong>所属组 (Group)</strong>: 文件或目录所属的用户组。
5.  <strong>大小 (Size)</strong>: 文件的大小（以字节为单位）。
6.  <strong>最后修改月份</strong>: ...
7.  <strong>最后修改日期</strong>: ...
8.  <strong>最后修改时间</strong>: ...
9.  <strong>名称 (Name)</strong>: 文件或目录的名称。

举个例子，如果我对<code>-</code>开头的文件使用cd命令，是会被拒绝的，因为这是一个普通文件而不是一个目录。

## 3. 硬链接 (Hard Link) vs 软链接 (Soft Link)

- <strong>Inode</strong>: 可以理解为文件在硬盘上的唯一标识符，存储了文件的元数据（如权限、大小、位置等）。

### 硬链接 (<code>ln</code>)
- <strong>创建</strong>: <code>ln <original_file> <hard_link_name></code>
- <strong>特性</strong>:
  - 实质上是给同一个 inode 创建了另一个文件名。
  - 删除、移动或重命名原始文件，不会影响硬链接，因为它直接指向 inode。
  - 不能跨文件系统创建。
  - 不能对目录创建硬链接。

### 软链接/符号链接 (<code>ln -s</code>)
- <strong>创建</strong>: <code>ln -s <original_file> <soft_link_name></code>
- <strong>特性</strong>:
  - 相当于一个快捷方式，存储的是原始文件的路径。
  - 如果原始文件被删除、移动或重命名，软链接将失效（dangling link）。
  - 可以跨文件系统创建。
  - 可以对目录创建软链接。

## 4. I/O 重定向与管道

Linux Shell中有三个标准流：
- 文件描述符<strong>0</strong>: <code>stdin</code> (标准输入)
- 文件描述符<strong>1</strong>: <code>stdout</code> (标准输出)
- 文件描述符<strong>2</strong>: <code>stderr</code> (标准错误)

### 重定向 (Redirects)
- <strong>输出重定向</strong>:
  - <code>></code>: 将命令的 <strong>标准输出</strong> 重定向到一个文件（覆盖文件原有内容）。<code>></code> 符号默认就是操作 <code>1 (stdout)</code>。
    - <code>ls -l > listings.txt</code>
    ![VirtualBoxVM_f7QGhtEPSz.png](https://pub-85d4dcece16844bf8290aa4b33608ccd.r2.dev/ShareX/2025/10/VirtualBoxVM_f7QGhtEPSz.png)
  - <code>>></code>: 将命令的 <strong>标准输出</strong> 追加到一个文件的末尾。
    - <code>echo "Hello" >> listings.txt</code>
    ![VirtualBoxVM_awiNI9DV83.png](https://pub-85d4dcece16844bf8290aa4b33608ccd.r2.dev/ShareX/2025/10/VirtualBoxVM_awiNI9DV83.png)
- <strong>错误重定向</strong>:
  - <code>2></code>: 将命令的 <strong>标准错误</strong> 重定向到一个文件。
    - <code>ls /nonexistent 2> error.log</code>
    ![VirtualBoxVM_AXNVY779w6.png](https://pub-85d4dcece16844bf8290aa4b33608ccd.r2.dev/ShareX/2025/10/VirtualBoxVM_AXNVY779w6.png)
- <strong>输入重定向</strong>:
  - <code><</code>: 将文件的内容作为命令的 <strong>标准输入</strong>。
    - <code>cat < listings.txt</code>
    ![VirtualBoxVM_2zVUYsvw02.png](https://pub-85d4dcece16844bf8290aa4b33608ccd.r2.dev/ShareX/2025/10/VirtualBoxVM_2zVUYsvw02.png)

### 管道 (Pipes)
- <strong>符号</strong>: <code>|</code>
- <strong>作用</strong>: 将一个命令的 <code>stdout</code> 直接连接到另一个命令的 <code>stdin</code>，实现命令的串联处理。
- <strong>示例</strong>:
  ``<code>bash
  # 将 ls -l 的输出通过 more 命令进行分页显示
  ls -l | more
  </code>``
  ![VirtualBoxVM_wzmqbbh4BI.png](https://pub-85d4dcece16844bf8290aa4b33608ccd.r2.dev/ShareX/2025/10/VirtualBoxVM_wzmqbbh4BI.png)

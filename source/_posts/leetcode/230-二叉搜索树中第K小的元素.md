# 230. 二叉搜索树中第K小的元素

> **题目链接：**[230. 二叉搜索树中第K小的元素](https://leetcode.cn/problems/kth-smallest-element-in-a-bst/)

## 题目描述

给定一个二叉搜索树的根节点 `root` ，和一个整数 `k` ，请你设计一个算法查找其中第 `k` 个最小元素（从 1 开始计数）。

## 核心思考：二叉搜索树的中序遍历特性

这是有关**二叉搜索树 (BST，Binary Search Tree)** 特性最基础也是最重要的灵魂知识考点：
**二叉搜索树天生就是一个排好序的数字阵列集合投影，只要我们对它进行“中序遍历（左 -> 根 -> 右）”，得到的访问输出结出阵列绝对是一个一毛不差的完美「升序排序序列」！**

明白了这个底气特性规律，题目就瞬间简单并暴力化：
无脑启动一盘标准的树探寻框架深入中序提取，把沿路顺带摘抄下来的所有沿途子节点通通扔进提前备好的蓄水列表缸子 `mid`。
挂机等程序遍历扫荡结束后，直接把列表以 `k-1` 为寻宝尺码拔下提取出那份宝藏并读它的内属核心面额传票值 `val` 就大满贯成功。

*(注：如果有极致性能追求强迫症希望提前终止截断剪枝的开发大佬，可以通过引入外部计录器挂坠的方式替代死存拉满数组。在遭遇摘录访问操作累跳值等同于设定 K 时立即鸣金收兵打断截击递归)*

## 解题思路 (Python)

```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def kthSmallest(self, root: Optional[TreeNode], k: int) -> int:
        mid = []
        
        def dfs(node):
            if node == None:
                return
            # 中序：极其纯正严格的 左->根->右
            dfs(node.left)
            mid.append(node)
            dfs(node.right)
        
        dfs(root)
        
        # 因为题目下标题意标的第 1 起始偏移计数特性习惯，所以寻库抓址需要往后抹平一分偏差 (-1)
        return mid[k-1].val
```

## 复杂度分析

- **时间复杂度**：O(N)，作为毫不保留压阵的暴走铺满图纸中序列搜方案，不管你的 K 偏前还是靠末期压底箱位置，都被它硬性铺展刷开一遍整树所有的网状节枝。
- **空间复杂度**：O(N)，构建了一支完整包含囊括网罗装载承接进树体全貌所有的宏硕辅阵巨型托盘清单，叠加算上原生态调用潜游下沉递归自带吃线暗亏，在劣态下同归消耗。

# 303. 区域和检索 - 数组不可变

> <strong>题目链接：</strong>[303. 区域和检索 - 数组不可变](https://leetcode.cn/problems/range-sum-query-immutable/)

## 题目描述

给定一个整数数组 <code>nums</code>，处理以下类型的多个查询：计算索引 <code>left</code> 和 <code>right</code>（包含 <code>left</code> 和 <code>right</code>）之间的 <code>nums</code> 元素的和，其中 <code>left <= right</code>。

实现 <code>NumArray</code> 类：
- <code>NumArray(int[] nums)</code> 使用数组 <code>nums</code> 初始化对象。
- <code>int sumRange(int left, int right)</code> 返回数组 <code>nums</code> 中索引 <code>left</code> 和 <code>right</code> 之间的元素的总和。

## 核心思考：前缀和

这道题的本质就是前缀和（Prefix Sum）。

如果每次调用 <code>sumRange</code> 都遍历区间 <code>[left, right]</code> 来累加求和，单次查询的时间复杂度为 $O(N)$。当查询次数很多时，这种做法效率很差。

前缀和的思路是：在初始化阶段预先计算好一个前缀和数组 <code>s</code>，其中 <code>s[i]</code> 表示原数组前 <code>i</code> 个元素的总和。这样，任何区间 <code>[left, right]</code> 的和都可以通过一次减法得到：

$$sumRange(left, right) = s[right + 1] - s[left]$$

这里 <code>s</code> 的长度比 <code>nums</code> 多 1，<code>s[0] = 0</code> 作为边界条件，避免了对 <code>left = 0</code> 的特殊处理。

## 解题思路 (Python)

``<code>python
class NumArray:

    def __init__(self, nums: List[int]):
        s = [0] * (len(nums) + 1)
        for i,x in enumerate(nums):
            s[i + 1] = s[i] + x
        self.s = s

    def sumRange(self, left: int, right: int) -> int:
        return self.s[right + 1] - self.s[left]


# Your NumArray object will be instantiated and called as such:
# obj = NumArray(nums)

## 复杂度分析

- <strong>时间复杂度</strong>：
  - 初始化：$O(N)$，其中 $N$ 是数组 </code>nums<code> 的长度。需要遍历一次数组来构建前缀和。
  - 单次查询：$O(1)$。每次调用 </code>sumRange` 只需要做一次减法运算。
- <strong>空间复杂度</strong>：$O(N)$。需要额外存储一个长度为 $N + 1$ 的前缀和数组。

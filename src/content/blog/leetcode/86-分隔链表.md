# 86. 分隔链表

> <strong>题目链接：</strong>[86. 分隔链表](https://leetcode.cn/problems/partition-list/)

## 题目描述

给你一个链表的头节点 <code>head</code> 和一个特定值 <code>x</code> ，请你对链表进行分隔，使得所有 <strong>小于</strong> <code>x</code> 的节点都出现在 <strong>大于或等于</strong> <code>x</code> 的节点之前。

你应当 <strong>保留</strong> 两个分区中每个节点的初始相对位置。

## 核心思考：双虚拟头节点构建法

本题最直观的解法是<strong>模拟法</strong>。
与其在原来的链表上进行复杂的节点移除和插入操作，不如直接建立<strong>两条全新的链表</strong>：
1. <strong>小链表 (<code>dummy1</code>)</strong>：专门用来接收所有值 <code>< x</code> 的节点。
2. <strong>大链表 (<code>dummy2</code>)</strong>：专门用来接收所有值 <code>>= x</code> 的节点。

我们只需遍历原链表，遇到小于 <code>x</code> 的节点将其链接到 <code>dummy1</code> 后，大于等于 <code>x</code> 的节点将其链接到 <code>dummy2</code> 后。遍历结束后，再将 <code>dummy1</code> 的尾部连接到 <code>dummy2</code> 的实际头部（即 <code>dummy2.next</code>），即可完成链表的分隔。

### 细节：处理链表成环（断尾）

在将原链表节点重组到新链表时，一个常见的错误是<strong>链表成环</strong>。这是因为节点挂载到新链表时，依然保留着原始的 <code>next</code> 指针。如果不加以处理，新链表的尾节点可能会指向链表中的某个前驱节点，从而导致结构成环。

解决这个问题，有两种常见的处理思路（代码中均有注释说明）：

<em> <strong>法 1：逐个断开 <code>next</code> 指针</strong>
  在将当前节点 <code>curp</code> 接入新链表之前，先利用临时变量将其下游 <code>curp.next</code> 暂存，随后立刻将 <code>curp.next</code> 置为 <code>None</code>。这样可以保证接入新链表的每个节点都是独立的，从根本上阻断了成环的可能。
</em> <strong>法 2：遍历结束后统一断开结尾</strong>
  在挂载节点的过程（如 <code>p1.next = curp</code>）中，大部分节点的 <code>next</code> 指针已经被重新指向了正确的下一个节点。因此在遍历中，我们可以直接推进 <code>curp = curp.next</code>，无需刻意断开原指针。<strong>然而</strong>，当遍历结束时，大链表的尾节点 <code>p2</code> 的 <code>next</code> 指针依旧保留着原始状态。如果它恰好指向了一个被分配至小链表中的节点，拼接两链表后便会成环。
  因此，使用法 2 的核心在于：<strong>在退出循环后、拼接大小链表之前，必须显式地执行一次 <code>p2.next = None</code>，强制断开尾节点的旧指针。</strong> 这种做法相较于法 1 更加简洁，也是在算法实现中更为标准的推荐写法。

## 解题思路 (Python)

``<code>python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
class Solution:
    def partition(self, head: Optional[ListNode], x: int) -> Optional[ListNode]:
        dummy1 = ListNode()
        dummy2 = ListNode()
        p1,p2 = dummy1,dummy2

        curp = head

        while curp:
            if curp.val < x:
                p1.next = curp
                p1 = p1.next
            else:
                p2.next = curp
                p2 = p2.next

            # 法1：清除目前curp.next指针的指向（因为在指向老的两个链表之一）
            # temp = curp.next
            # curp.next = None
            # curp = temp

            # 法2：最后连接前清除p2的尾巴
            curp = curp.next
        p2.next = None

        p1.next = dummy2.next # 比x小的和比x大的连起来

        return dummy1.next
</code>`<code>

## 复杂度分析

- <strong>时间复杂度</strong>：$O(n)$，其中 $n$ 是原链表的长度。我们仅仅对原链表进行了一次线性无回溯的遍历。
- <strong>空间复杂度</strong>：$O(1)$。虽然我们“建立”了两条新链表，但实际上只是额外声名了几个虚拟节点 (</code>dummy1<code>, </code>dummy2<code>, </code>p1<code>, </code>p2`) 的指针变量，那些庞大的节点内存全部都是原地复用的原链表节点对象，并没有申请 $O(n)$ 的新内存。

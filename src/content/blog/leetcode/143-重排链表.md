# 143. 重排链表

> <strong>题目链接：</strong>[143. 重排链表](https://leetcode.cn/problems/reorder-list/)

## 题目描述

给定一个单链表 <code>L</code> 的头节点 <code>head</code> ，单链表 <code>L</code> 表示为：

<code>L0 → L1 → … → Ln-1 → Ln</code>

请将其重新排列后变为：

<code>L0 → Ln → L1 → Ln-1 → L2 → Ln-2 → …</code>

不能只是单纯的改变节点内部的值，而是需要实际的进行节点交换。

## 核心思考：利用栈结构逆序连接

链表不支持随机访问，无法直接获取尾部节点。为了实现首尾交替的重排，我们可以利用栈（Stack）后进先出（LIFO）的特性。

1. <strong>第一次遍历</strong>：将链表的所有节点按顺序压入栈中。这使得我们可以通过不断弹栈（<code>pop</code>）的方式，逆序获取链表的尾部节点。
2. <strong>第二次遍历与重组</strong>：使用指针 <code>p</code> 从头节点开始遍历。每次循环提取当前节点的下一个节点 <code>next_node</code>，并从栈顶弹出一个尾部节点 <code>last_node</code>。
3. 把弹出的 <code>last_node</code> 插入到 <code>p</code> 和 <code>next_node</code> 之间。
4. 指针 <code>p</code> 随后移动到 <code>next_node</code>，为下一组交替插入做准备。

### 细节：重组的终止条件

链表重排最关键的地方在于判断何时停止拼装，避免形成环路。这取决于链表节点总数的奇偶性：

- <strong>当节点总数为偶数时</strong>：前后两个方向的指针最终会紧挨在一起。此时栈顶弹出的节点 <code>last_node</code> 刚好等于原链表中 <code>p</code> 的下一个节点 <code>next_node</code>。
- <strong>当节点总数为奇数时</strong>：前后两个方向的指针最终会指向同一个中间节点。即栈顶弹出的 <code>last_node</code> 与当前的 <code>p</code> 是同一个节点。这也意味着此时 <code>last_node.next == next_node</code>。

当满足以上任意一种情况（<code>last_node == next_node or last_node.next == next_node</code>）时，说明首尾交替已经触达到链表的中心区域。此时必须断开循环，并将 <code>last_node.next</code> 显式置为 <code>None</code>，以封闭链表尾部，防止在此处继续循环引用从而产生死循环。

## 解题思路 (Python)

``<code>python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
class Solution:
    def reorderList(self, head: Optional[ListNode]) -> None:
        """
        Do not return anything, modify head in-place instead.
        """
        stack = []
        p = head
        while p is not None:
            stack.append(p)
            p = p.next
        
        p = head
        while p is not None:
            last_node = stack.pop()
            next_node = p.next
            if last_node == next_node or last_node.next == next_node:
                last_node.next = None
                break
            p.next = last_node
            last_node.next = next_node
            p = next_node
</code>``

## 复杂度分析

- <strong>时间复杂度</strong>：$O(N)$，其中 $N$ 是链表的长度。需要遍历链表两次，一次压栈，一次重组。
- <strong>空间复杂度</strong>：$O(N)$。使用了额外的数组（栈）结构存储所有节点引用。

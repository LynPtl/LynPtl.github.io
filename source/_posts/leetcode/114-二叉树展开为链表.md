# 114. 二叉树展开为链表

> **题目链接：**[114. 二叉树展开为链表](https://leetcode.cn/problems/flatten-binary-tree-to-linked-list/)

## 题目描述

给你二叉树的根结点 `root` ，请你将它展开为一个单链表：
- 展开后的单链表应该同样使用 `TreeNode` ，其中 `right` 子指针指向链表中下一个结点，而左子指针始终为 `null` 。
- 展开后的单链表应该与二叉树 **先序遍历** 顺序相同。

## 核心思考：带着引线开展前序穿梭编织

我们要按照“先序遍历（根 -> 左 -> 右）”的规矩去拉直这棵庞大的繁根木须藤，强行将其梳理揉直成为单列直立光秃杆子（右枝相连引长）。

棘手的麻烦和冲突点在哪里？
一旦在遍历访问处理父节点那当刻，极其狠毒果敢绝决地把当前 `curr.right` 桥梁掰弯斩首重新架设向下一个新挂位落脚点的时候，原先老家挂在那里等下要深潜勘测下去那条右系藤蔓分支，就随风在虚空宇宙里**永远丧失迷失引带源丢弃**无法逆向拾回被生生腰斩回收内存垃圾了（或者造成污染死环死锁死跌入逻辑黑洞）！

最稳妥保险最干净的做法：
1. **备档与寄存**：在打算做手术砍切改造某棵子节点 `curr` 之前，把它的 `left` 与 `right` 各自原本通往谁的数据先通过局部保险柜死库保存抢救下来！
2. **重塑引流与桥接改造**：将外部引入伴随牵线的一个长外链随行老手 `curr` 游击探针将其左钩挂除锁死卸空 `left = None`，并把挂接拉挂的尾套钩绳 `right = node` 死死钉接嵌实过去目标上并滑进接位推前继续随同向下一方前行待命就位。
3. **安全下沉**：因为原始桥被改造但预设藏好的老桥寄存安全锁保险图还在，所以此时毫无恐惧阴云困扰，坦荡肆意用拿回原预备好寄留下的新安全信标沿着备置线路正常走往探索下阶段进程继续推线前进深水池区域不坠渊底。

## 解题思路 (Python)

```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def flatten(self, root: Optional[TreeNode]) -> None:
        """
        Do not return anything, modify root in-place instead.
        """
        if not root:
            return
        curr = dummy = TreeNode()
        # 前序遍历接入链表
        def dfs(node):
            nonlocal curr
            
            if node == None:
                return

            # 第一步核心关键逃命寄生舱保护！抢救提取预防污染覆盖冲刷篡改
            left_node = node.left
            right_node = node.right

            # 第二步实施肆无忌惮毫无顾忌放心魔改指针打砸抢串联工程拉伸拔长梳理
            curr.left = None
            curr.right = node
            curr = node

            # 第三步取出拿回上供封存妥善收保遗留真迹放还引导重回探墓旅途
            dfs(left_node)
            dfs(right_node)

        dfs(root)
        return dummy.right

```

## 复杂度分析

- **时间复杂度**：O(N)。整棵巨繁茂林子无论盘根错节繁复到什么夸张规模阶下都只能乖乖接受从前到后只刮行一遍秋风扫落叶大探底式走廊地毯清洗勘访，无二刷内耗时间拖延折损拖沓时间浪费。
- **空间复杂度**：O(N)。在树高深度坠下的调用呼号队列由于未进行重组在原始骨干修剪切切开路属于原生纯朴直接潜层推入，且受无极限链长化隐式深不见底连累影响可能会暴逼触碰顶格天花上限。

# 160. 相交链表

> <strong>题目链接：</strong>[160. 相交链表](https://leetcode.cn/problems/intersection-of-two-linked-lists/)

## 题目描述

给定两个单向链表的头节点 <code>headA</code> 和 <code>headB</code>，请找出并返回两个单向链表相交的起始节点。如果不存在交点，则返回 <code>null</code>。

## 核心思考：双指针消除长度差

这道题最直观的难点在于，两个链表的长度可能不一致，因此我们无法让两个指针从头开始同步走到相交点。

如果能想办法<strong>消除长度差</strong>，让两个指针处于同一起跑线上，问题就迎刃而解了。如何巧妙地做到这一点呢？我们可以让两个指针各自走过对方走过的路：

1. <strong>逻辑上的路径拼接</strong>：假设链表 A 的独立长度为 <code>a</code>，链表 B 的独立长度为 <code>b</code>，两者相交的公共部分长度为 <code>c</code>。
2. 指针 <code>p1</code> 从 <code>headA</code> 出发，走完链表 A 后，立刻跳转到 <code>headB</code> 的开头继续走。它走过的总路程将是：<code>a + c + b</code>。
3. 指针 <code>p2</code> 从 <code>headB</code> 出发，走完链表 B 后，立刻跳转到 <code>headA</code> 的开头继续走。它走过的总路程将是：<code>b + c + a</code>。
4. <strong>殊途同归</strong>：显而易见，<code>a + c + b = b + c + a</code>。无论两条链表原本有多长，只要按照这种方案走，两个指针最终一定会<strong>同时到达相交节点</strong>。

那如果两个链表压根不相交呢？同理，当指针分别走过 <code>a + b</code> 和 <code>b + a</code> 的距离后，它们会同时指向各自尽头的空指针 <code>None</code>，也就是在末尾相遇并双双退出。

## 解题思路 (Python)

我们只需要用 <code>while p1 != p2</code> 作为跳出条件，一旦某个指针走到尽头，就让它从另一个链表的头部重新开始。

``<code>python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, x):
#         self.val = x
#         self.next = None

class Solution:
    def getIntersectionNode(self, headA: ListNode, headB: ListNode) -> Optional[ListNode]:
        p1,p2 = headA,headB
        while p1 != p2:
            if p1 == None:
                p1 = headB
            else:
                p1 = p1.next
            if p2 == None:
                p2 = headA
            else:
                p2 = p2.next
        return p1
</code>``

## 复杂度分析

- <strong>时间复杂度</strong>：$O(m+n)$，其中 $m$ 和 $n$ 分别为两个链表的长度。两个指针最差情况下各自遍历一遍链表 A 和链表 B，总体执行次数线性相关。
- <strong>空间复杂度</strong>：$O(1)$。算法只分配了两个额外的指针指向已有节点，没有任何多余的内存开辟。

# 118. 杨辉三角

> **题目链接：**[118. 杨辉三角](https://leetcode.cn/problems/pascals-triangle/)

## 题目描述

给定一个非负整数 `numRows`，生成「杨辉三角」的前 `numRows` 行。
在「杨辉三角」中，每个数是它左上方和右上方的数的和。

## 核心思考：双层迭代层层堆叠

杨辉三角问题看似吓人，但它其实根本无需什么深奥的数论算法（排列组合也可以做，但是杀鸡不用牛刀）。
这是一道非常原汁原味考验代码数组二维矩阵编织构造基本盘掌控力的“纯正找规律模拟题”。

基本构造法则规律极其固定死板：
1. 每一层金字塔行的行内元素总数都跟本身目前所处于的阶段高度行数是相呼应一一对应的（第 `i` 层刚好拥有 `i+1` 号元素长位坑）。
2. 每层队伍排头的首号人物（`j=0`）以及最终收尾的末端防守大将（`j=i`）被天然锁死永远被钦定为数字 `1`。
3. 夹夹在内部区间的各个安全避风港坑位 `j` 他们的成型产物值，完全依赖挂靠在它们头顶肩上侧上方一代前一层楼 `i-1` 数据网系。等同于吸纳传承上方偏左角 `[i-1][j-1]` 及上方正脑门直角 `[i-1][j]` 原值的汇合累加合并馈赠。

## 解题思路 (Python)

```python
class Solution:
    def generate(self, numRows: int) -> List[List[int]]:
        ans = []
        for i in range(numRows):
            line = [1] * (i + 1)
            line[0] = line[i] = 1
            if i >= 2:
                for j in range(1,i):
                    # 取出上方左上和右上两个位置相加
                    line[j] = ans[i-1][j] + ans[i-1][j-1]
            ans.append(line)
        return ans
```

## 复杂度分析

- **时间复杂度**：O(N^2)，这里 N 象征指定的 `numRows` 参数指标层级规模总量。通过代码非常直观可见一枚外围包裹式主控遍历轮与内部核心执行补齐缝隙循环交织结合；需要运算处理的填数模块呈现标准的逐渐膨胀梯形等差面积公式加和级数。
- **空间复杂度**：O(1)（如果不计要抛出去打包返还的解答体系集合数组的话），其执行纯粹在内部提取过往遗留原位档案进行查账抄录不另拉山头开辟超纲储备盘。如果计算存储返回数组则为 O(N^2)。

# 169. 多数元素

> **题目链接：**[169. 多数元素](https://leetcode.cn/problems/majority-element/)

## 题目描述

给定一个大小为 `n` 的数组 `nums` ，返回其中的多数元素。多数元素是指在数组中出现次数 **大于** `⌊ n/2 ⌋` 的元素。
你可以假设数组是非空的，并且给定的数组总是存在多数元素。

## 核心思考：Boyer-Moore 摩尔投票法

当然可以简单粗暴地用字典或者排序后直接取中位数，但最高阶也是限制要求中最希望考察的方法是**摩尔投票法 (Boyer-Moore Voting Algorithm)**，它可以在极致的 O(N) 时间 + O(1) 空间下大杀四方。

投票法的核心就是**“拼死对子抵消”**机制：
假设有多个黑帮火拼，其中一个名叫“多数派”的黑帮人数**比其他所有小派系加起来还要多**。
规则很简单：不同阵营的人一旦相遇，就直接一换一同归于尽（抵消扣分）。由于“多数派”阵营本身足够庞大过半，哪怕所有散杂阵营全部联手集火它一个，最后拼了个精光，场上唯一能剩下存活下来的，**绝对必将是多数派的残党**。

- 维护一个擂主 `ans` 和擂主的当前生命血量计数 `times`。
- 如果血槽空了 `times == 0`：下一个出现的人就直接窃取阵地强行上位成新擂主。
- 如果不是空槽：判断来者是不是擂主的同伙（`x == ans`），是同伙就合兵一处 `times += 1`，要是遭遇异族直接拔刀互拼掉一管血 `times -= 1`。
扫图结尾最终停留在总宝座上的那张面孔，即为多数元素。

## 解题思路 (Python)

```python
class Solution:
    def majorityElement(self, nums: List[int]) -> int:
        ans = times = 0
        for x in nums:
            if times == 0:
                ans, times = x, 1
            else:
                if x == ans:
                    times += 1
                else:
                    times -= 1
        return ans

```

## 复杂度分析

- **时间复杂度**：O(N)，一盘从头扫到尾的流水线作战。
- **空间复杂度**：O(1)，抛弃传统哈希计票改用实时结算，仅用到两个整数常驻指标项内存开销。
